## Request

Reimplement the currently Clojure-shaped Walter language leaves as real language-native packages:

- `/home/ubuntu/code/bigconfig/walter/typescript` in TypeScript
- `/home/ubuntu/code/bigconfig/walter/python` in Python

`walter/clojure` is the reference implementation.

## Current findings

- `walter/python` and `walter/typescript` are currently near-copies of the Clojure leaf: they still have `deps.edn`, `bb.edn`, Clojure source under `src/clj`, Clojure tests, and Clojure-oriented `CLAUDE.md` / `README.md`.
- The Clojure reference now exposes the launcher-friendly `run` UX:
  - `package validate|build|create|delete`
  - individual tools: `tofu`, `ansible`, `ansible-local`
- Walter's package workflow is intentionally smaller than Once:
  - `create`: Once `tofu` -> Walter `ansible` -> Once `ansible-local`
  - `build`: render those three stages
  - `delete`: Once `tofu` destroy only
- Walter owns only the remote Ansible tool/data/rendering layer. It reuses Once's `tofu`, `ansible-local`, and Tofu-output parameter extraction.
- Walter validation has already been added in Clojure and should be ported, including schema, tool, live credential, R2, ssh-agent, and Walter Ansible-data checks.
- The Walter Clojure source had unused legacy `src/resources/.../walter/tools/tofu/{hcloud,oci}` templates. They are not rendered by the current workflow and have been removed from the Clojure reference as part of this plan.
- For launcher compatibility, each non-Clojure leaf must have exactly one language manifest at the repo root (`pyproject.toml` or `package.json`) and a root `run` file. Leaving `deps.edn` in those leaves would make language detection ambiguous or wrong.
- TypeScript target packages need committed `dist/` output if the launcher-installed `run` file imports `walter/dist/src/cli.js`, mirroring `once/typescript`.

## Decisions from review

1. **Artifact parity:** Python and TypeScript `package build` output must be byte-for-byte identical to the Clojure reference artifact.
2. **CLI shape:** mirror `once/python` and `once/typescript`:
   - Python exposes a console script, e.g. `uv run walter -- package build`, plus root `run` for launcher use.
   - TypeScript exposes an npm dev script/bin, e.g. `npm run walter -- package build`, plus root `run` for launcher use.
3. **Dependency mode:** committed manifests pin `once` and `big-config` by Git SHA. Local workspace paths may be used only as temporary development overrides. The final committed BigConfig SHA is aligned with the pinned Once dependency in each language so dependency resolution/deduplication succeeds. Relevant workspace paths:
   - Python BigConfig: `/home/ubuntu/code/bigconfig/big-config/python`
   - TypeScript BigConfig: `/home/ubuntu/code/bigconfig/big-config/typescript`
4. **TypeScript dist:** commit `walter/typescript/dist/` like `once/typescript/dist/` so GitHub-installed launcher runs do not require a postinstall build.
5. **Unused Walter Tofu resources:** remove the legacy Walter-owned `tools/tofu` templates from the Clojure reference and do not port them to Python/TypeScript. Walter uses Once's shared `tofu` workflow/templates.
6. **Safety:** do not run destructive/provisioning commands unless explicitly approved.

## Removed legacy Walter Tofu resources

The Clojure Walter source tree contained:

- `src/resources/io/github/bigconfig-ai/walter/tools/tofu/hcloud/main.tf`
- `src/resources/io/github/bigconfig-ai/walter/tools/tofu/oci/main.tf`

These templates were not rendered by the current Clojure workflow:

- `bb run tofu ...` dispatches to Once's shared `io.github.bigconfig-ai.once.tools/tofu` workflow.
- `package build/create/delete` also call Once's `tofu` workflow.
- Walter's own `tools.clj` defines only the Walter `ansible` workflow.

They were legacy/source-only templates from before Walter reused Once's shared Tofu stage. They did not affect the current `.dist` artifact, which renders Once's `tofu` templates plus Walter's Ansible templates.

Decision: remove them from `walter/clojure` and omit them from the Python/TypeScript ports.

## Standing assumptions

- Preserve the Clojure Walter CLI/workflow surface exactly; do not add Once-only commands such as `describe`, `tofu-smtp`, `tofu-dns`, or a top-level `validate` shortcut.
- Byte-for-byte build artifact parity with Clojure is required.
- Commit language-native manifests/locks and remove Clojure manifests from the Python/TypeScript leaves.
- Use Git-SHA dependencies in committed manifests; avoid committed local path dependencies.
- Keep generated `.dist/` out of source and never edit it manually.

## Proposed implementation plan

### 1. Establish the language-native package skeletons

For `walter/python`:

- Remove Clojure-only root files from the active package shape: `deps.edn`, `bb.edn`, `src/clj`, `test/clj`, Clojure caches/config that are no longer useful.
- Add `pyproject.toml` for package `walter`, Python >= 3.12, console script `walter = "walter.cli:main"`, Git-SHA dependency pins for `big-config` and `once`, pytest dev dependency, and resources included in wheels/sdists.
- Add `src/walter/` modules plus `py.typed`.
- Add `tests/` pytest suite.
- Add/update `run`, `README.md`, `CLAUDE.md`, `.gitignore`, `.envrc`, `.envrc.private.example`, and `devenv.nix` for Python + uv + Ansible + OpenTofu.

For `walter/typescript`:

- Remove Clojure-only root files from the active package shape: `deps.edn`, `bb.edn`, `src/clj`, `test/clj`, Clojure caches/config that are no longer useful.
- Add `package.json` for package `walter`, ESM, Node >= 20, bin `walter`, Git-SHA dependency pins for `big-config` and `once`, dev deps (`typescript`, `tsx`, `vitest`, `@types/node`).
- Add `tsconfig.json`, `vitest.config.ts`, `src/cli.ts`, `src/walter/*.ts`, and `test/*.test.ts`.
- Add/update `run`, `README.md`, `CLAUDE.md`, `.gitignore`, `.envrc`, `.envrc.private.example`, and `devenv.nix` for Node/npm + Ansible + OpenTofu.
- Build and commit `dist/` for launcher compatibility, mirroring `once/typescript`.

### 2. Port common Walter concepts from Clojure

Implement equivalent modules in both languages:

- `interop` helpers mirroring Once's Python/TypeScript alias bridge for `profile`, `params`, `exit`, and `err` while preserving BigConfig namespaced keys.
- `options` with the Walter profile (`walter`) and active profile (`bb`), using safe placeholder values where public defaults should not embed credentials.
- `params`:
  - `opts_fn` / `optsFn`: `readBcPars` then Once `tofu_params` / `tofuParams`.
  - `walter_opts` / `walterOpts`: apply deterministic prefix `io.github.bigconfig-ai.walter.package/start-create-or-delete` before `opts_fn`.
- Keep kebab-case parameter names at the boundary (`provider-compute`, `compute-pubkey`, `r2-secret-access-key`, etc.).

### 3. Port Walter Ansible data/rendering

Create `ansible.py` and `ansible.ts` equivalent to `walter/clojure/src/clj/.../ansible.clj`:

- `data_fn` / `dataFn` builds hosts, users, config, repos, and devbox packages.
- Use `compute-pubkey` as `config.ssh_key`, matching the Clojure validation work.
- Generate:
  - `packages.yml`
  - `repos.yml`
  - `ssh-config.yml`
  - `inventory.json`
  - `default.config.yml`
- Match Clojure's generated formatting byte-for-byte for files produced by Walter's render functions (YAML/JSON/string quoting/order/newlines).

### 4. Port Walter tools

Create `tools.py` and `tools.ts` with:

- Walter `ANSIBLE` workflow key: `io.github.bigconfig-ai.walter.tools/ansible`.
- Shared step functions: print step, exit step, print-error step.
- Template rendering for Walter-owned Ansible resources:
  - base copy from `.`
  - generated `roles/users/tasks/{packages,repos,ssh-config}.yml`
  - generated `{inventory.json,default.config.yml}`
- `ansible_star` / `ansibleStar` CLI-ready wrappers.
- Reuse Once's `tofu` and `ansible-local` implementations rather than reimplementing them.
- Add package-resource path resolution where needed (especially TypeScript; Python can mirror Once but must work from source and launcher installs).

### 5. Port the package workflow

Create `package.py` and `package.ts`:

- Register subworkflows for:
  - Once `TOFU`
  - Walter `ANSIBLE`
  - Once `ANSIBLE_LOCAL`
- Implement:
  - `create`: `tofu ["render tofu:init tofu:apply:-auto-approve", opts_fn]`, `ansible ["render ansible-playbook:main.yml", opts_fn]`, `ansible-local ["render ansible-playbook:main.yml", opts_fn]`
  - `build`: same stages with `"render"`
  - `delete`: Once `tofu ["render tofu:init tofu:destroy:-auto-approve", opts_fn]`
- Wire `validate` through BigConfig's `VALIDATE_FN` / `WF_VALIDATE_FN`.
- Do not wire `describe` unless a future plan asks for it.
- Return/sync alias keys consistently for CLI/tests.

### 6. Port validation

Create `validation.py` and `validation.ts`, based on the Clojure Walter validation and the existing Once Python/TypeScript validation style:

- Pure report builder: `validate_report` / `validateReport`.
- Workflow step: `validate` prints grouped output and sets non-zero exit on failure.
- Schema checks for:
  - profile/package/compute-pubkey
  - compute providers: `oci`, `hcloud`, `digitalocean`, `no-infra`
  - backends: `s3`, `r2`, `local`
  - optional `compute-prevent-destroy`
- Tool checks:
  - base: `tofu`, `ansible-playbook`, `ssh`, `curl`
  - provider/backend: `oci`, `hcloud`, `doctl`, `aws`
- Credential checks:
  - Hetzner, DigitalOcean, OCI, S3, R2
  - cloud compute ssh-agent check for `compute-pubkey`
  - skip ssh-agent for `no-infra`
- Walter Ansible-data checks:
  - hosts present and real
  - active users include `ubuntu`
  - `config.ssh_key` matches `compute-pubkey`
  - repos/packages/users have required fields
- Keep side effects behind injectable seams (`which`, `run`, env) so tests do not need real credentials or tools.

### 7. Port the CLI and launcher `run` files

Python:

- `src/walter/cli.py` with help text and `main(argv=None, opts=None)`.
- Support:
  - `package <validate|build|create|delete>...`
  - individual `tofu`, `ansible`, `ansible-local`
  - help/unknown-command behavior matching Clojure.
- Root `run` Python script loads local `.venv` / `src` like `once/python/run`, defines safe `DEFAULT_PROFILE`, and calls `walter.cli.main`.

TypeScript:

- `src/cli.ts` with `main(argv, opts = bb)` and the same commands.
- Root `run` Node script loads built local `dist/src/cli.js` or package import, defines safe `DEFAULT_PROFILE`, and calls `main`.
- Help text should make launcher usage clear and avoid Clojure-specific `bb run` wording.

### 8. Resources and docs

- Move/copy Walter-owned templates to:
  - Python: `src/resources/io/github/bigconfig-ai/walter/tools/ansible/...`
  - TypeScript: `src/resources/io/github/bigconfig-ai/walter/tools/ansible/...`
- Omit the removed legacy Walter `tools/tofu` resource trees; Walter uses Once's shared `tofu` resources.
- Update README/CLAUDE per leaf with language-specific commands:
  - Python: `uv sync`, `uv run pytest -q`, `uv run walter -- ...`, `uv run python run ...`
  - TypeScript: `npm install`, `npm run typecheck`, `npm test`, `npm run build`, `npm run walter -- ...`, `node run ...`
- Document that `.dist/` is generated and that destructive provisioning commands require explicit approval/real credentials.

### 9. Tests

Python pytest and TypeScript Vitest should cover:

- CLI help and command routing, including no top-level `validate` shortcut if we preserve Clojure behavior.
- `package validate` workflow step success/failure exit behavior via injected report builders.
- Schema validation with stub credentials for all compute/backend combinations.
- Placeholder/missing parameter failures.
- Provider tool selection and injected `which` behavior.
- R2 error classification.
- ssh-agent checks for cloud compute and skip for `no-infra`.
- Ansible data/rendering output for inventory, config, packages, repos, and ssh-config.
- `package build` non-destructive render test comparing `.dist` byte-for-byte against the Clojure reference artifact.
- Root `run` default profile shape stays free of real credentials.
