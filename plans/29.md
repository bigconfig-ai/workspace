## Goal

Update the root `bb run git` helper so local commits that have not been pushed to the repository's upstream remote are first-class in both reporting and commit/push automation.

Desired behaviour:

- `bb run git report` shows repos with unpushed commits, including repos whose working tree is otherwise clean.
- `bb run git report --porcelain` exposes unpushed-commit state in machine-readable output.
- `bb run git commit` processes:
  - dirty repos, as today, with the existing `pi ... "commit and push"` command; and
  - clean repos that have one or more commits ahead of their upstream remote, with direct `git push`.
- `bb run git commit --dry-run` previews the same expanded set and command choice.

## User decisions

1. Clean repos with unpushed commits should use direct `git push`, not `pi ... "commit and push"`.
2. Repos with no configured upstream should show a visible `no upstream` warning in reports.
3. Repos that are both ahead and behind/diverged should be reported and skipped by `git commit` rather than processed automatically.
4. `--porcelain` may add new labels such as `UNPUSHED` and `DIRTY-UNPUSHED`.

## Working assumptions

- Use each repo's upstream branch (`@{upstream}`) to determine pushed/unpushed status.
- Compute divergence with `git rev-list --left-right --count @{upstream}...HEAD`:
  - left count = remote commits not local (`behind`);
  - right count = local commits not remote (`ahead` / unpushed).
- Treat `ahead > 0` and `behind = 0` as safe to process:
  - if the working tree is clean, run direct `git push`;
  - if the working tree is dirty, run the existing `pi ... "commit and push"` command.
- Treat `ahead > 0` and `behind > 0` as diverged: show it in `git report`, and in `git commit` print a skip/report row instead of running either `pi` or `git push`.
- Show behind-only counts in `git report`; behind-only repos do not trigger `git commit` unless they are also dirty.
- Show visible `no upstream` warnings in both human and porcelain reports. Repos with no upstream cannot be classified as pushed/unpushed, so `git commit` should not run direct `git push` for them; dirty no-upstream repos still use the existing `pi` path.
