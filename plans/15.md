# 15 — Rewrite root git workspace helper in Babashka

## Request

Rewrite `/home/ubuntu/code/bigconfig/git-dirty-report.sh` in Babashka and rename it to a clearer name, because the current script can do more than report: it can also ask `pi` to commit and push dirty repositories.

## Current behavior to preserve unless changed

- Default root: `/home/ubuntu/code/bigconfig`.
- Finds nested git repositories by searching for `.git` directories or files under the root.
- Default subcommand: `report`.
- Supported subcommands:
  - `report`: list clean/dirty/no-tracked-files repos and print a summary.
  - `commit`: for each dirty repo, run `pi --print --model deepseek-v4-flash 'commit and push'`.
- Supported options:
  - `--porcelain` for machine-readable report output.
  - `--dry-run` for commit preview.
  - `--help` / `-h`.
- Positional parsing currently accepts either `[subcommand] [root-dir]` or `[root-dir]`.
- Dirty details are based on `git status --porcelain`.
- Repos with no tracked files are reported but skipped by commit mode.

## Decisions

1. New script name: `run`.
2. Preserve the current CLI exactly, including positional root parsing and option names.
3. Keep the default root hardcoded to `/home/ubuntu/code/bigconfig`.
4. Commit mode must keep using exactly `pi --print --model deepseek-v4-flash 'commit and push'`; do not add prompt/model flags or environment overrides.
5. If a `pi` commit command fails, stop immediately.
6. Delete `git-dirty-report.sh`; do not keep a compatibility wrapper.
7. `--porcelain` does not need byte-for-byte compatibility, but should remain stable and machine-parseable.

## Proposed target

- Replace the Bash implementation with a Babashka script named `run`, using only standard Babashka/Clojure facilities plus shelling out to `find`, `git`, and `pi` as needed.
- Keep behavior compatible except that `--porcelain` may differ in formatting if the new format remains stable and machine-parseable.
- Use a direct executable script with shebang `#!/usr/bin/env bb`.
- Remove the old `git-dirty-report.sh` after the Babashka replacement is verified.

## Implementation plan

1. Create the new Babashka script at `run`.
2. Port argument parsing, help text, repo discovery, status collection, report output, and commit dispatch while preserving the existing CLI.
3. Make command execution explicit and testable with small pure helpers where reasonable:
   - parse args
   - classify porcelain status lines
   - format human/porcelain rows
   - collect repo metadata
4. Preserve deterministic repo ordering.
5. Ensure dirty/no-tracked/clean counts match the chosen compatibility behavior.
6. Set executable bit on `run`.
7. Delete `git-dirty-report.sh`.

## Verification

- `bb run --help`
- `bb run report`
- `bb run --porcelain`
- `bb run commit --dry-run`
- Optional comparison before deletion: compare Bash and Babashka `report` output on the same checkout.
