# 16 — Reshape root `run` CLI around command groups

## Request

Change `/home/ubuntu/code/bigconfig/run` so it exposes a grouped interface:

- `bb run` → show top-level help.
- `bb run git` → show git command help.
- `bb run git report` → report dirty git repositories.
- `bb run git commit` → commit and push dirty repositories.
- Rename the Babashka namespace from `git-workspace-run` to `run`.

The executable `./run` can continue to work, but user-facing help and examples should advertise only `bb run ...`.

## Current state

- `run` already exists as an executable Babashka script.
- Current namespace: `git-workspace-run`.
- Current CLI is git-only:
  - `./run` / `bb run` defaults to `report`.
  - `./run report` reports dirty/clean repositories.
  - `./run commit` runs `pi --print --model deepseek-v4-flash 'commit and push'` in each dirty repo.
- Current supported options:
  - `--porcelain` for report mode.
  - `--dry-run` for commit mode.
  - `--help` / `-h`.
- Current positional parsing accepts `[subcommand] [root-dir]` or `[root-dir]`.
- Default root remains `/home/ubuntu/code/bigconfig`.

## Proposed target behavior

### Top-level help

- `bb run`
- `bb run --help`
- `bb run -h`

Should show top-level help that lists available command groups, currently:

- `git` — git workspace report / commit helpers.

### Git help

- `bb run git`
- `bb run git --help`
- `bb run git -h`

Should show git-specific help with:

- `bb run git report [options]`
- `bb run git commit [options]`

Common git option:

- `--root-dir DIR` — scan repositories under `DIR` instead of `/home/ubuntu/code/bigconfig`.

### Git report

- `bb run git report` reports dirty/clean repositories under the default root.
- `bb run git report --root-dir DIR` reports repositories under `DIR`.
- Preserve existing report behavior and formatting unless intentionally changed.
- Preserve `--porcelain` exactly.
- `--dry-run` is invalid for report.

### Git commit

- `bb run git commit` commits and pushes dirty repositories using the existing `pi` command.
- `bb run git commit --root-dir DIR` commits dirty repositories under `DIR`.
- Preserve existing behavior of stopping immediately if `pi` fails.
- Preserve `--dry-run` exactly.
- `--porcelain` is invalid for commit.

## Observations from current `run`

- `--help` / `-h` currently prints one flat help screen and short-circuits parsing wherever it appears.
- Flags may appear before or after positionals. Command-specific flags are parsed globally, so `--dry-run` is ignored by report and `--porcelain` is ignored by commit.
- `./run foo` currently treats `foo` as a root directory, not as an unknown subcommand.
- Extra positionals are not reported as errors by the old parser.

## Decisions

1. No flat compatibility aliases: `bb run report` and `bb run commit` should fail with an error plus top-level help.
2. Help text and examples should advertise only `bb run ...`.
3. Keep configurable root support, but replace positional `[root-dir]` with `--root-dir DIR`.
4. Keep `--porcelain` and `--dry-run` with their exact current names and behavior.
5. Treat command-specific flags on the wrong command as validation errors.
6. Treat extra positionals as validation errors.
7. For unknown commands/options, print an error plus the relevant help and exit `1`.

## Implementation plan

1. Rename namespace in `run` to `(ns run ...)`.
2. Replace the flat argument parser with a two-level parser:
   - top-level dispatcher: currently only `git`, plus top-level help and unknown-group handling.
   - git dispatcher: `report`, `commit`, git help, and unknown-subcommand handling.
3. Change the default no-argument behavior from report to top-level help.
4. Add separate top-level and git help text advertising only `bb run ...`.
5. Keep the existing repo discovery, metadata collection, report output, and commit dispatch functions.
6. Replace positional root parsing with `--root-dir DIR`.
7. Remove flat `report` / `commit` compatibility behavior.
8. Validate command-specific flags and extra positionals as errors.
9. Verify the executable bit remains set.
