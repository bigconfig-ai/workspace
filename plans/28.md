## Content outline

### 1. Package-author skills (`#package-author-skills`)

Explain:

- BigConfig publishes Claude Code skills for scaffolding new BigConfig packages.
- The skills live at <https://github.com/bigconfig-ai/skills>.
- Each skill creates a minimal, real BigConfig package that:
  - depends only on the `big-config` SDK;
  - exposes `package validate`, `package describe`, `package build`, `package create`, and `package delete`;
  - renders one safe `hello` stage;
  - ships a launcher-friendly root `run` file;
  - is ready to be launched by `bc-pkg` after pushing to GitHub.

### 2. Available skills

Document only these three:

| Skill | Language | Use when |
|---|---|---|
| `create-ts-bigconfig-package` | TypeScript | Create a new npm/Node BigConfig package. |
| `create-py-bigconfig-package` | Python | Create a new PyPI/uv BigConfig package. |
| `create-clj-bigconfig-package` | Clojure | Create a new deps.edn/Babashka BigConfig package. |

### 3. Install/list with `npx skills` (`#package-author-skills-install`)

Include short commands:

```bash
# List the BigConfig skills
npx skills add bigconfig-ai/skills -l --full-depth

# Install all three skills for Claude Code globally
npx skills add bigconfig-ai/skills \
  --global \
  --agent claude-code \
  --skill create-ts-bigconfig-package create-py-bigconfig-package create-clj-bigconfig-package \
  --yes

# Or install one skill
npx skills add bigconfig-ai/skills \
  --global \
  --agent claude-code \
  --skill create-ts-bigconfig-package \
  --yes
```

Mention that project-local installs are also possible by omitting `--global` from inside a project.

### 4. Use with Claude (`#package-author-skills-use`)

Show an example that generates a prompt from a skill and sends it to Claude without requiring prior installation:

```bash
npx skills use bigconfig-ai/skills \
  --skill create-ts-bigconfig-package \
  --agent claude-code
```

Also show a pipe-oriented example for the `claude` CLI:

```bash
npx skills use bigconfig-ai/skills@create-ts-bigconfig-package | claude
```

Then give example prompts after installing the skills:

```text
/create-ts-bigconfig-package scaffold a package named analytics-worker for owner bigconfig-ai
/create-py-bigconfig-package scaffold a package named analytics-worker for owner bigconfig-ai
/create-clj-bigconfig-package scaffold a package named analytics-worker for owner bigconfig-ai
```

### 5. Shared scaffold workflow (`#package-author-skills-workflow`)

Present the shared step-by-step flow:

1. Pick the language skill: TypeScript, Python, or Clojure.
2. Provide package name, GitHub owner, description, target directory, and SDK source.
3. The skill derives language-specific names/tokens.
4. The skill copies templates into the target package and substitutes tokens.
5. The skill runs the language's install/test/lifecycle commands.
6. The skill reports the generated package checklist and how to grow the package.

Keep this concise; do not duplicate every template file mapping from each `SKILL.md`.

### 6. Language-specific gotchas

Include a compact note/card list:

- TypeScript: package name may have hyphens, but `__PKG_VAR__` must become a valid JS identifier; imports use explicit `.js` extensions.
- Python: `__PKG_MODULE__` is snake_case and distinct from the PyPI/CLI package name.
- Clojure: source paths munge hyphens to underscores, while resource paths keep hyphens.
- All languages: boundary/template params stay kebab-case; no real credentials; `.dist/` is generated output.

### 7. Verify generated packages (`#package-author-skills-verify`)

Include concise command blocks:

```bash
# TypeScript
npm install
npm run build
npm run typecheck
npm test
node run package validate
node run package describe
node run package build
node run package create
node run package delete
BC_PAR_NAME=REPLACE_ME node run package validate  # expected to fail
```

```bash
# Python
uv sync
uv run pytest -q
uv run python run package validate
uv run python run package describe
uv run python run package build
uv run python run package create
uv run python run package delete
BC_PAR_NAME=REPLACE_ME uv run python run package validate  # expected to fail
```

```bash
# Clojure
clojure -M:test
bb run package validate
bb run package describe
bb run package build
bb run package create
bb run package delete
BC_PAR_NAME=REPLACE_ME bb run package validate  # expected to fail
```

Expected result:

- normal lifecycle commands exit 0;
- the `BC_PAR_NAME=REPLACE_ME ... validate` command exits 1;
- `build` / `create` render `.dist/<name>-<hash>/.../hello/greeting.txt` containing `Hello, world!`.
