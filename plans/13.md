3. Update docs/manuals and historical plan files too (`README.md`, `CLAUDE.md`, `index.html`, `changes.html`, `plans/*.md`, etc.). Target: zero legacy-prefix references in editable source/docs.
4. Regenerate and commit `once/typescript/dist/` after source changes.

6. Include root workspace docs/manuals if they mention the legacy prefix; scope is not limited to the four listed leaves for documentation cleanup.
