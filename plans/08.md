# Goal

Clean up obsolete files in the launcher repositories after the `bc-pkg` launcher
split:

- `launcher/typescript` should be the npm package.
- `launcher/python` should be the PyPI package.

# Rationale

The Python launcher is now implemented as a Python package under
`launcher/python/src/bc_pkg` with `launcher/python/pyproject.toml`. It no longer
needs the old Node/Babashka launcher scaffolding.

The TypeScript launcher is still the npm package, so it must keep
`package.json`, `bin/bc-pkg.js`, and `README.md`, but it no longer needs the old
Babashka/Docker/manual files.

# Resolved decisions

- Remove historical `plans/` directories from both launcher repos.
- Delete local ignored `launcher/python/.venv/` during cleanup.
- Do not keep a Python `bin/bc-pkg` compatibility shim. The only Python package
  entrypoint is the `pyproject.toml` console script:

```toml
[project.scripts]
bc-pkg = "bc_pkg.cli:main"
```
