1. Naming style:
   - first relevant mention: **BigConfig SDK**;
   - subsequent generic mentions: **the SDK**;
   - language-specific mentions: **Python SDK**, **TypeScript SDK**, **Clojure SDK**.
2. Scope includes **every source documentation file in the workspace**, not only the three SDK leaf projects.
3. Do **not** update historical documents.
   - Exclude changelogs, old plan files, old blog/social drafts, generated session transcripts, and similar archival content.
   - Exception: update talk HTML pages under `once-bigconfig/main/src/pages/talk/**`.
4. In `once-bigconfig/main/src/pages/manual.html`, update only references to the library/engine. Keep broader BigConfig ecosystem/brand references unchanged.
5. Keep **ÔÇťBigConfig packageÔÇŁ** as the term for Once/Walter-style packages. They are packages that use the SDK.
6. Do not update package metadata descriptions outside docs, such as `pyproject.toml`, `package.json`, `deps.edn`, lockfiles, generated metadata, or badges.
7. Package/repo/import/install identifiers need case-by-case handling in docs. Because metadata is out of scope, do not invent or apply new package coordinates in this plan. Keep code blocks and commands functional, and rewrite surrounding prose to use the SDK naming.

## Review of the original request
