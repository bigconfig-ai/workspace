# 12 — Walter Clojure namespace rename + Once/BigConfig + launcher compatibility

## Request

Change `/home/ubuntu/code/bigconfig/walter/clojure` so it can develop against:

- `/home/ubuntu/code/bigconfig/once/clojure`
- `/home/ubuntu/code/bigconfig/big-config/clojure`

Also add a root `run` script similar to `/home/ubuntu/code/bigconfig/once/clojure/run`, align Walter's Clojure package shape closely enough that it works with `/home/ubuntu/code/bigconfig/launcher/typescript` (`bc-pkg`), and rename Walter's namespace prefix from `io.github.amiorin.walter` to `io.github.bigconfig-ai.walter`.

## Review notes

- `walter/clojure` currently pins old Git deps and requires old Once namespaces (`io.github.amiorin.once.*`). The local `once/clojure` package uses `io.github.bigconfig-ai.once.*` and coord `io.github.bigconfig-ai/once`, so Walter source imports must change before local Once can be used.
- `walter/clojure` has no root `run` file. The TypeScript launcher requires a target repo to have exactly one language manifest at the root (`deps.edn` for Clojure) and a root `run` file that can be copied into the caller's cwd.
- `once/clojure/run` is a Babashka script that defines a safe default profile and delegates command parsing to a CLI namespace. Walter should mirror this pattern rather than putting all launcher behavior in `bb.edn` tasks.
- Renaming `io.github.amiorin.walter` to `io.github.bigconfig-ai.walter` means Clojure source file paths should move from `src/clj/io/github/amiorin/walter/...` to `src/clj/io/github/bigconfig_ai/walter/...`. Resource paths likely need the same move because `keyword->path` derives template paths from namespace-qualified keywords.
- The launcher synthesizes the Clojure dependency coordinate from the target GitHub slug. For `bigconfig-ai/walter@clojure`, callers will depend on `io.github.bigconfig-ai/walter`, so Walter's own `bb.edn` self-coordinate should use the same org/name.
- There is a launcher-compatibility tension: committed `:local/root` deps such as `../../once/clojure` or `/home/ubuntu/...` will not work when `bc-pkg` installs Walter from GitHub into a user's machine. For `bc-pkg`, Walter's committed `deps.edn` must use Git/Maven coordinates for transitive deps; local roots should be dev-only overrides or temporary workspace edits.
- `bb.edn` currently contains task shortcuts, but the target shape is Once-like dependency config only; `bb run ...` should execute the root `run` file.

## Decisions

1. Use Git SHAs in `deps.edn` for both normal use and development. Do not commit local-root dependency overrides.
2. Test launcher compatibility against `bigconfig-ai/walter@clojure`.
3. Expose the Once-style CLI shape through `run`: `bb run package create`, `bb run package build`, `bb run tofu render`, etc.
4. Add a non-destructive `build` step that renders Walter's stages without applying/provisioning.
5. The new root `run` file must use safe `REPLACE_ME` placeholders, not the current OCI/S3 defaults.
6. Do not keep the old `bb` tasks as supported shortcuts; switch the UX to `bb run ...`.
7. Rename Walter's namespace prefix from `io.github.amiorin.walter` to `io.github.bigconfig-ai.walter`.

## Namespace rename decisions

1. Make the namespace change as a hard rename only; do not keep compatibility aliases/shims for old `io.github.amiorin.walter.*` namespaces.
2. Move the resource root too, from `src/resources/io/github/amiorin/walter/...` to `src/resources/io/github/bigconfig-ai/walter/...`, so `keyword->path` template lookup remains consistent with renamed namespaces.
3. Use `io.github.bigconfig-ai/walter` as the `bb.edn` self/package coordinate, matching `bc-pkg`'s synthesized coordinate for `bigconfig-ai/walter@clojure`.
4. Public docs should only mention the new `io.github.bigconfig-ai.walter` namespace; do not document the old namespace.
5. Keep generated profile/package names as `walter` (`.dist/walter-...`, `:package "walter"`) after the namespace rename.

## Proposed implementation plan

1. **Namespace and package-coordinate rename**
   - Move Clojure namespaces from `io.github.amiorin.walter.*` to `io.github.bigconfig-ai.walter.*`.
   - Move source files from `src/clj/io/github/amiorin/walter/` to `src/clj/io/github/bigconfig_ai/walter/`.
   - Move resource files from `src/resources/io/github/amiorin/walter/` to `src/resources/io/github/bigconfig-ai/walter/`.
   - Update all internal requires, qualified keywords, `workflow/new-prefix` values, comments, docs, and self dependency coordinates to the new namespace/package prefix.
   - Do not add compatibility shim namespaces for the old `io.github.amiorin.walter.*` prefix.
   - Keep runtime profile/package names as `walter`.

2. **Dependency alignment**
   - Update Walter's Once dependency/imports from `io.github.amiorin.once.*` to `io.github.bigconfig-ai.once.*`.
   - Update `deps.edn` to use Git SHA dependencies for `io.github.bigconfig-ai/once` and `io.github.amiorin/big-config`; do not use local-root overrides.
   - Verify the selected SHAs expose the APIs Walter needs and are compatible with `bc-pkg` installation from GitHub.

3. **Add a Walter CLI namespace**
   - Add `src/clj/io/github/bigconfig_ai/walter/cli.clj`, modeled after `once/clojure/src/.../cli.clj`.
   - Support help output, `package` workflow commands, and individual tool commands.
   - Canonical UX is `bb run package <build|create|delete>` and `bb run <tool> ...`; do not preserve old `bb walter ...` task shortcuts.

4. **Add root `run` script**
   - Create executable `walter/clojure/run` with a safe placeholder default profile and `(cli/main* *command-line-args* default-profile)`.
   - Use the same namespaced keys as Once (`::render/profile`, `::workflow/params`) and rely on `BC_PAR_*` overrides for real values.

5. **Align workflow surface**
   - Keep create/delete behavior using Once's shared `tofu` and `ansible-local` tools plus Walter's own `ansible` tool.
   - Add a `build` workflow that renders `tofu`, `ansible`, and `ansible-local` without applying anything.
   - Clean up obsolete task-only code while preserving generated `.dist/` as generated output only.

6. **Docs and task config**
   - Update `README.md` / `CLAUDE.md` command examples to the canonical `bb run ...` UX.
   - Simplify `bb.edn` to dependency config only, mirroring Once; no `:tasks` shortcuts.

7. **Verification**
   - Non-destructive local checks:
     - `cd walter/clojure && bb run help`
     - `cd walter/clojure && bb run package build` (if build is added)
     - `cd walter/clojure && bb run tofu render`
     - `cd walter/clojure && clojure -M:test` (currently no test files expected)
   - Launcher check after changes are available on the chosen GitHub ref:
     - From a temp directory, run `node /home/ubuntu/code/bigconfig/launcher/typescript/bin/bc-pkg.js bigconfig-ai/walter@clojure package build`.
   - Do not run `create`, `delete`, `tofu:apply`, or `tofu:destroy` unless explicitly approved.
