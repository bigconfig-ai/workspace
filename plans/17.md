# 17 — `bc-pkg` must support local repositories too

Status: implemented in this checkout (both launchers).

Verification run (all from a fresh scratch directory, against the local
`once/{clojure,python,typescript}` leaves):

- `bc-pkg ../once/python package build` → `.dist/default-378cc184/` (Python launcher).
- `bc-pkg ../once/typescript package build` → `.dist/default-378cc184/` (TS launcher).
- `bc-pkg ../once/clojure package build` → `.dist/default-378cc184/` (via Python launcher).
- Live-edit check: editing `once/python/src/once/cli.py` is picked up on the next
  run with no re-init and no rebuild.
- Both launchers emit **byte-identical** `deps.edn`/`bb.edn`/`package.json`/
  `pyproject.toml` and identical `run` symlinks for the same local path.
- Re-entry: same path = no-op; different local path, local↔GitHub switch, and
  GitHub spec in a local dir all hard-fail.
- cwd guard: `bc-pkg . ...` from inside a leaf is refused; the leaf is left clean.

## Request

Extend the `bc-pkg` launcher so it can target a **local repository** on disk, not
only a GitHub `<owner/repo@ref>`. This must land in both launcher
implementations (`launcher/python`, `launcher/typescript`) as a single,
behaviour-parity change set.

## Current state

`bc-pkg` is GitHub-only today. Both launchers share the same shape:

- **Spec parsing** (`parse_spec` / `parseSpec`): `SPEC_RE = ^([^/\s@]+)/([^/\s@]+)@([^\s]+)$`
  — strictly `owner/repo@ref`. Anything else falls through to forwarded args.
- **Ref resolution** (`resolve_ref` / `resolveRef`): full 40-char SHA passes
  through; otherwise `GET /repos/{owner}/{repo}/commits/{ref}` → `data.sha`.
- **File fetch** (`fetch_file` / `fetchFile`): pulls `deps.edn`, `package.json`,
  `pyproject.toml`, and `run` from the pinned SHA via the GitHub contents API.
- **Target detection** (`detect_target` / `detectTarget`): exactly one of the
  three manifests must exist; that selects the language. TS/Python read the
  package name from the manifest; Clojure synthesises `io.github.{owner}/{repo}`.
- **Manifest writing**: pins a **git** dependency —
  Clojure `:git/url` + `:git/sha` (deps.edn + bb.edn), TS `github:owner/repo#sha`,
  Python `pkg @ git+https://github.com/owner/repo.git@sha` — plus a `bigconfig`
  metadata block (`repo`, `ref`, `sha`, `language`, `run`, `packageName`).
- **Re-entry** (`read_metadata` + `validate_existing_metadata`): a mismatch in
  repo/ref/sha is a hard error, never an implicit upgrade.
- **Run** (`run_target` / `runTarget`): TS `npm install` then `node run`;
  Python `uv sync` then `uv run python run`; Clojure bootstraps Babashka + JDK
  then `bb run`.

There is no code path that reads from the local filesystem instead of GitHub.

### Constraints carried from the launcher CLAUDE.md files

- **No runtime dependencies** — Python stdlib only / Node built-ins only.
- **On-disk artifact parity** between the two launchers — mirror every change.
- **Do not import** from `big-config` / `once` / `selmer`.
- **No implicit auto-upgrade** of an initialised directory.

## Primary use case

**Live local dev against the sibling leaves.** A developer in a scratch
directory wants to point `bc-pkg` at a working copy of a local target package
(e.g. `../once/clojure`) and have their uncommitted edits picked up on the next
run — no GitHub round-trip, no SHA pinning, no rebuild-and-push loop.

## Decisions

1. **Live working copy, not a snapshot.** Local targets resolve to native
   local-path dependency coordinates so edits flow through immediately
   (subject to each toolchain's build step — see caveats below).

2. **Invocation syntax — normal local path as the first arg.** The first arg is
   treated as a local target when it is path-like: it begins with `/`, `./`,
   `../`, or `~`. Such values cannot match `SPEC_RE` (`owner/repo@ref`) and will
   not collide with bare forwarded command words like `package` (no leading
   slash/dot). A bare relative name with no path prefix is **not** treated as a
   path, to avoid misreading forwarded args. (No `file:` input scheme — plain
   paths only; the `file:` form is reserved for the generated npm dependency.)
   - `bc-pkg ../once/clojure package validate`
   - `bc-pkg /abs/path/to/once/python package validate`
   - `bc-pkg ~/code/bigconfig/once/typescript package validate`

3. **Native local-path dependency form:**
   - **Clojure** → `bb.edn` (and `deps.edn`) use `:local/root "/abs/path"`
     instead of `:git/url` + `:git/sha`.
   - **TypeScript** → `package.json` dependency `"file:/abs/path"` (npm symlinks
     the local dir).
   - **Python** → `pyproject.toml` keeps the dep name in `[project.dependencies]`
     and adds a `[tool.uv.sources]` editable path source
     (`pkg = { path = "/abs/path", editable = true }`).

4. **Metadata: path is the identity; no `ref`/`sha`.** The `bigconfig` block
   gains a local marker plus the resolved absolute path:
   - Clojure → `:bigconfig/local "true"` and `:bigconfig/path "/abs/path"`.
   - TS → `bigconfig.local = true`, `bigconfig.path = "/abs/path"`.
   - Python → `local = true`, `path = "/abs/path"` under `[tool.bigconfig]`.
   `repo`/`ref`/`sha` are omitted for local targets; `read_metadata`'s
   completeness check branches on the local marker (requires `path` + `language`
   instead of `repo`/`ref`/`sha`).

5. **Re-entry validation compares the resolved absolute path.** Same path =
   proceed to run (no re-init). Different path, or a previously git-pinned dir,
   = hard error (matching today's strict no-implicit-upgrade rule). Switching a
   local dir to a GitHub spec (or vice-versa) is likewise a hard error.

6. **Refuse to overwrite the leaf in place.** If the resolved local path equals
   cwd, fail — `bc-pkg` would otherwise clobber the leaf's own
   `deps.edn`/`package.json`/`pyproject.toml`. Direct the user to run from a
   separate scratch directory.

7. **`run` file is symlinked (copy fallback).** For live dev the `run` entry
   point is symlinked from the local target into cwd so run-file edits flow
   through, falling back to a copy where symlinks are unavailable
   (mirrors `_expose_python_resources`). GitHub targets keep copying as today.

8. **Scope:** all three target languages, both launchers, one change set, no new
   runtime deps. Relative paths resolve against cwd; `~` expands. The target dir
   must contain exactly one manifest (same one-language rule as `detect_target`,
   reused by reading from disk instead of the GitHub contents API).

## Caveats to document (not blockers)

- **TypeScript** local dev still requires the local leaf to be built (its
  `dist/`), because `file:` resolves to the package's published entry points —
  same as the GitHub path, which relies on the package's `prepare`/build.
- **Python** editable installs surface `resources/` from the source tree, so
  `_expose_python_resources` must also handle the editable layout (resources in
  the source dir, not under `.venv/.../site-packages`).
- **Clojure** `:local/root` is genuinely live (Babashka loads source directly).

## Proposed implementation plan

Mirror every step across `launcher/python/src/bc_pkg/cli.py` and
`launcher/typescript/bin/bc-pkg.js`.

1. **Local spec detection.** Add an `is_local_spec` / `isLocalSpec` test
   (path-prefix `/`, `./`, `../`, `~`), and a `LocalSpec` value carrying the resolved
   absolute path (cwd-relative resolution + `~` expansion). `parse_spec` /
   `parseSpec` returns either a GitHub `Spec` or a `LocalSpec`.
2. **Local file reads.** Add a `read_local_file` helper used by a local-aware
   `detect_target` and `initialize`, replacing the GitHub `fetch_file` calls for
   local targets. Reuse the existing exactly-one-manifest rule.
3. **Manifest writers.** Branch each `write_*_manifest` on local vs. git:
   emit `:local/root` / `file:` / `[tool.uv.sources]` editable, and the local
   metadata marker + `path`.
4. **Metadata model + validation.** Extend `Metadata` with `local` + `path`;
   relax the completeness check for local; teach `validate_existing_metadata`
   to compare paths and reject local↔git transitions.
5. **`run` file.** Add a symlink-with-copy-fallback path for local targets;
   keep GitHub copy/restore behaviour. Restore logic reads from the local path
   when `meta.local`.
6. **cwd guard.** Reject local path == cwd before writing any manifest.
7. **Python resources.** Make `_expose_python_resources` find `resources/`
   in the editable source tree as well as site-packages.
8. **Docs/parity.** Update both `CLAUDE.md` (Behaviour Contract + Parity table
   + What to Avoid) and both `README.md` with the local-repo usage and caveats.
